redis常用的数据结构之skiplist

发表于 2018-09-19 | 分类于 redis | 阅读次数:

介绍

redis常用的数据结构skiplist，及其底层数据结构实现。

版本：>= 4.0

数据结构

redis的zset是一个复合结构，

  • 需要hash存储value和score的关系

  • 还需要score排序，指定score范围查找，跳跃链表

思路

  • 最底层双向链表，保证数据存储

  • 概率分层结构保证二分查找，范围查找

  • redis分层64层

查找，插入，更新

  • 二分查找

  • 逐层下降，搜索路径

随机层数：每插入一个节点，需要随机一个合理的层数，level1是50%，level2是25%

插入数据

  • 查找到位置，如果score相等判断value

  • 创建节点，分配随机层数，

  • 将搜索路径上的节点和新节点通过前后指针串起来

删除数据：类似插入数据

更新：先删除，再插入

排名计算：每个forward指针增加span，表示当前层的forward跳到当前节点有多少个节点。将搜索路径想加即可。
