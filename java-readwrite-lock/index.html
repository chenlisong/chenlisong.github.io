Java读写锁的实现

简单实现

方法定义，不管是简单实现还是进阶实现，读写锁暴露四个方法

  • public synchronized void lockRead()

  • public synchronized void unlockRead()

  • public synchronized void lockWrite()

  • public synchronized void unLockWrite()

且都是synchronized的方法，因为其在实例级别并发下，需要保持先后顺序。

读写锁的要求：

  • lockRead时，不能有写，写等待请求。

  • unlockRead时，无要求

  • lockWrite时，不能有读，写请求

  • unlockWrite时，无要求

高阶实现

上述实现中未解决几个问题。

  • 线程一unlock了线程二的锁

  • 锁的可重入性

总结一下要求

  • lockRead时，

    1. 如果当前线程正在写入，可以读

    2. 如果有其他线程正在写入，不可以读

    3. 如果当前线程正在读，可以读

    4. 如果有其他写等待请求，不可以读

    5. 其他情况，可以读

  • unlockRead时

    1. 判断是否当前线程正在持有该lock

  • lockWrite时，不能有读，写请求

    1. 如果只有当前线程读，可以写

    2. 有其他线程读，不可以写

    3. 没有写入线程，可以写

    4. 不是当前线程写，不可以写

  • unlockWrite时，

    1. 不是当前线程，抛异常
