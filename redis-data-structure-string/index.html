redis常用的数据结构之string

发表于 2018-09-18 | 分类于 redis | 阅读次数:

介绍

redis常用的数据结构string，及其底层数据结构实现。

涉及的内容：string, list, hash, set, zset, hyperloglog, rebloom, redis-cell

版本：>= 4.0

RedisObject

redis对象头的数据结构

一个RedisObject占用16个bytes。

embstr & raw

string的存储结构。

string长度小于44时，采用的紧凑string结构。

embstr将RedisObject对象头和sds连续存储，只需要一次malloc，占用内存更少。但是每次改变需要重新分配内存。

raw需要两次malloc，两个对象头在内存地址不连续。

因为jemalloc分配内存的单位是2的指数。如果长度超过64bytes，redis认为是大字符串。不再使用embstr存储，改用raw形式。

看上面这张图可以算出，留给 content 的长度最多只有 45(64-19) 字节了。字符串又是以\0结尾，所以 embstr 最大能容纳的字符串长度就是 44。

string-keys

  • 字符串最大长度512m

  • 创建字符串len和capacity一样长

  • 长度小于1m，扩容*2；超过1m，每次扩容+1m
