redis常用的数据结构之dict

发表于 2018-09-19 | 分类于 redis | 阅读次数:

介绍

redis常用的数据结构dict，及其底层数据结构实现。用于hash。

版本：>= 4.0

数据结构

  • hash结构使用dict

  • redis所有的key-value组成一个全局dict

  • 带过期时间的key也是一个dict

  • zset存储value和score值映射关系使用dict

dict有两个hashtable，通常只有一个有值，但是在扩容缩容时，会触发渐进式搬迁。搬迁结束后，旧的被删除。

类似Java的HashMap，数组+链表的形式。

渐进式rehash

大字典的扩容是遍历链表中所有旧元素，copy到新的数组下，O(n)的操作，所以redis执行渐进式搬迁。

  • 当前dict的hset，hdel会触发搬迁操作

  • redis定时任务中主动进行搬迁

其他

扩容场景

  • 正常情况下，hash表元素个数等于数组长度时，扩容*2

  • 如果上述情况，redis正在bgsave，为了减少内存页的过多分离，redis不会做rehash dict。除非元素个数等于数组长度的5倍，强制扩容

缩容

  • 元素个数小于一维数组长度，缩容

  • 如果上述场景正在bgsave，除非元素个数达到数组长度的10%，强制缩容

bgsave：fork一个子进程来创建rdb文件，并将内容数据写入到rdb中，父进程继续处理请求命令。内存*2
